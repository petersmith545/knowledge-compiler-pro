# 反向工程提示词

## 核心任务

理解B为什么更好，提炼出可复用的模式。

## 分析维度

| 对比维度 | 要回答的问题 | 提取目标 |
|---------|-------------|---------|
| 速度 | B为什么更快？ | 并行策略/缓存/简洁Prompt |
| 准确度 | B的输出为什么更准？ | Few-shot/二次验证/Schema约束 |
| 鲁棒性 | B遇到错误怎么处理？ | 重试机制/降级方案/异常捕获 |
| 输出质量 | B的格式为什么更好？ | 模板设计/后处理/约束指令 |
| Prompt策略 | B的指令有什么高明之处？ | CoT/分步指引/角色设定 |
| 工具使用 | B调用了什么不同的工具？ | 更好的API/脚本自动化 |

## 输出格式



markdown
反向工程报告: B → A

发现的优势模式
模式1: {名称}
来源: B的哪个部分
表现: 测试中带来了什么改善（量化）
原理: 为什么这样做更好
移植方案: 怎么应用到A上
风险评估: 可能的副作用
模式2: {名称}
...
